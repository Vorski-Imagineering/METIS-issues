# Django Integration for Multi-Agent Genesis Brain

## Architecture Overview

```
┌─────────────────────────────────────────────────────────────────┐
│                    Django Provisioning Layer                    │
│  ┌─────────────┐  ┌─────────────┐  ┌────────────────────────┐ │
│  │ Agent CRUD  │  │ Person ↔    │  │ SurrealDB Namespaces   │ │
│  │ (Admin/API) │  │ Agent Link  │  │ + Credentials          │ │
│  └─────────────┘  └─────────────┘  └────────────────────────┘ │
└─────────────────────┬───────────────────────────────────────────┘
                      │ API/JWT
┌─────────────────────┴───────────────────────────────────────────┐
│                  Hermes Agent Instances                         │
│  ┌─────────────┐  ┌─────────────┐  ┌────────────────────────┐ │
│  │ Personal    │  │ Community   │  │ Project                │ │
│  │ SQLite      │  │ SurrealDB   │  │ SurrealDB + ACLs       │ │
│  │ (private)   │  │ (shared)    │  │ (collaborative)        │ │
│  └─────────────┘  └─────────────┘  └────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

## 6. Security Model

- JWT tokens scoped to specific namespaces (agent can read their namespace + `tcc` shared)
- Tokens expire (30 days default, renewable via dashboard)
- Admin can revoke/regenerate tokens
- Personal data never shared between agents
- Community writes require PR-style approval flow

---

## 7. Key Workflows

### Agent Onboarding
1. Admin creates Agent in Django (or self-service via dashboard)
2. Django creates SurrealDB namespace + JWT token
3. Django generates `agent_config.yaml`
4. Hermes agent uses config to connect with scoped credentials

### Community KB Contribution
1. Agent writes to their namespace first
2. Django endpoint `/api/provision/kbsuggest/` accepts PRs
3. Admin reviews and merges to `tcc:knowledge_base`

### Offline Agent Handling
- Store last sync timestamp in Django
- Dashboard shows "stale" agents
- On reconnect, agent pulls recent community updates via API

## 10. Open Questions

1. Should each agent get their own SurrealDB instance or shared DB with namespaces?
   - **Recommended**: Shared DB with namespaces (simpler ops, sufficient isolation)

2. How to handle agent identity when people change Telegram handles?
   - **Recommended**: Link via Person record, use person.id as stable identifier

3. What's the approval workflow for community KB contributions?
   - **Recommended**: Django PR-style via `/api/provision/kbsuggest/` endpoint

4. Should agents automatically ingest their conversations into personal KB?
   - **Recommended**: Opt-in per agent, controlled by `knowledge_mode` field
